W: Duplicate sources.list entry http://us.archive.ubuntu.com/ubuntu/ trusty-updates/restricted amd64 Packages (/var/lib/apt/lists/us.archive.ubuntu.com_ubuntu_dists_trusty-updates_restricted_binary-amd64_Packages)
W: Duplicate sources.list entry http://us.archive.ubuntu.com/ubuntu/ trusty-updates/restricted i386 Packages (/var/lib/apt/lists/us.archive.ubuntu.com_ubuntu_dists_trusty-updates_restricted_binary-i386_Packages)
W: You may want to run apt-get update to correct these problems

Actually, you can't correct these problems by running $ sudo apt-get update

Solution: http://askubuntu.com/questions/120621/how-to-fix-duplicate-sources-list-entry

# backup
$ sudo mv /etc/apt/sources.list /etc/apt/sources.list.bak

# Alt+F2, type software-properties-gtk  
#
  •  Select all four catagory in Ubuntu Software tab
  • Select precise-security, precise-updates and precise-backports catagory in Updates tab.
  • Select Canonical partner and Intependent catagory from Other software tab.

# edit fresh sources.list
$ sudo vim /etc/apt/sources.list or $ sudo gedit /etc/apt/sources.list if you do not know how to use vim
# commend two lines in my computer
# deb http://us.archive.ubuntu.com/ubuntu/ trusty restricted
# deb http://us.archive.ubuntu.com/ubuntu/ trusty-updates restricted
(these two lines are duplicated with another two line in my computer:
deb http://us.archive.ubuntu.com/ubuntu/ trusty main restricted universe
deb http://us.archive.ubuntu.com/ubuntu/ trusty-updates main restricted universe)

# see the results
$ sudo apt-get update
